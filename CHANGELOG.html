Klen 3.0

April 2026 · Build 16

✦ A new name. The same obsession with simplicity.

Plink is now Klen — [klɛn]. Inspired by Scandinavian precision, the name reflects what this app has always been: the leanest way to capture tasks in the heat of the moment. No clutter, no friction. Just the essence of what needs to be done.

What's new

  • New New name & identity — Plink is now Klen. New icon, new name, same speed.
  • New Brand-new app icon — A clean, modern icon that reflects Klen's minimal character.
  • New About section in Settings — Name, pronunciation [klɛn], slogan, description, and version number — all in the Advanced tab.
  • New Reset Onboarding — Revisit the intro from Settings → Advanced anytime.
  • New Unit test suite — 60+ tests covering parser logic, dashboard sections, recurrence, data model edge cases, and purge behavior.

Improved

  • Improved App icon in Onboarding — The welcome screen now shows the real app icon instead of a generic symbol.
  • Improved Bundle ID & data paths — Updated to com.simonlang.Klen. Data stored under Klen/ in Application Support.
  • Improved All visible strings updated — Every user-facing mention of "Plink" replaced with "Klen" across English and German.
  • Improved Backup files — Now saved as .klenbackup instead of .plinkbackup.
  • Improved Sparkle update feed — Points to the new klen repository.

Bug fixes

  • Fix Token parser edge case — Input starting with a token symbol (e.g. @today Buy milk) now documented as known limitation; title correctly empty rather than silently swallowing text.
  • Fix Purge safety — Tasks soft-deleted without deletedAt set no longer risk being purged on next launch based on stale createdAt date.
  • Fix Recurrence edge case — Monthly recurrence from Jan 31 no longer crashes; Swift Calendar handles the date gracefully.
  • Fix Completed task visibility — Searching for a completed task correctly bypasses the 30-minute visibility window.

Noted in a blink. Organized in Klen.
